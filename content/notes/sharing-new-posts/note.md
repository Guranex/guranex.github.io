# Sharing new posts on X

After a new article is live, I use a short post format so sharing stays consistent.

## Recommended flow

1. Push the new article to GitHub
2. Wait for GitHub Pages to finish deploying
3. Open the live article URL
4. Post a short summary on X with the link

## Suggested post template

```text
I published a new post: <title>

What it covers:
- point 1
- point 2
- point 3

Read it here:
<url>
```

## What usually performs better

- A strong first sentence
- One clear takeaway
- The direct article URL
- One screenshot or code snippet if relevant
- Two or three focused tags, not too many

## Good examples of hooks

- I wrote up how I built my GitHub Pages blog.
- New post: notes from a recent web security lab.
- I documented the exact workflow I use to publish technical articles.

Consistency matters more than perfect wording, so the goal is to make sharing easy enough that I do it every time.
